# Checklist

Use this checklist to assess whether an organization has a minimum viable AI governance setup.

AI compliance is not a document, it is a system.

Within the SimpleAct framework, this checklist is not separate from the framework. It mirrors the five core components defined in [`framework.md`](./framework.md): inventory, classification, governance, documentation, and monitoring.

## Inventory

- all AI systems and use cases are listed
- each system has an owner
- key vendors and model dependencies are documented
- the system inventory can be reviewed and updated continuously

See [`templates/ai-system-inventory-template.md`](./templates/ai-system-inventory-template.md).

## Classification

- prohibited practice screening exists
- high-risk relevance is assessed
- transparency duties are considered
- reasoning and assumptions are recorded
- classification results can be reassessed when the system changes

See [`knowledge-base/eu-ai-act/risk-logic.md`](./knowledge-base/eu-ai-act/risk-logic.md) and [`templates/risk-classification-template.md`](./templates/risk-classification-template.md).

## Governance

- roles and approvals are defined
- escalation paths exist
- human oversight responsibilities are assigned
- there is a responsible function for ongoing review

See [`framework/control-domains.md`](./framework/control-domains.md).

## Documentation

- technical documentation exists for relevant systems
- decisions and evidence are retained
- templates are used consistently
- exportable records can be produced for audits or internal reviews

See [`templates/technical-documentation-template.md`](./templates/technical-documentation-template.md).

## Monitoring and Reporting

- incidents and issues are tracked
- material changes trigger reassessment
- review cadence is defined
- monitoring outputs are documented

See [`templates/post-market-monitoring-template.md`](./templates/post-market-monitoring-template.md).
