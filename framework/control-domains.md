# Control Domains

The framework is organized around the following control domains.

## 1. Governance and Accountability

- defined ownership
- approval and escalation paths
- policy alignment

## 2. AI System Inventory

- system registry
- use case mapping
- provider and model dependency capture

## 3. Risk Classification

- prohibited practice screening
- high-risk relevance assessment
- limited-risk transparency checks
- residual risk documentation

## 4. Documentation and Evidence

- technical documentation
- decision logs
- testing records
- vendor and contractual evidence

## 5. Human Oversight and Controls

- review checkpoints
- fallback procedures
- user and operator guidance

## 6. Monitoring and Change Management

- incident handling
- post-deployment review
- retraining or model-change assessments
- periodic control refresh
