# Main Content

This repository is the SimpleAct reference implementation for AI governance.

AI compliance is not a document, it is a system.

## What It Covers

- how to structure an AI governance system
- how to align that system with EU AI Act requirements
- how to turn governance work into repeatable records and processes

Within the SimpleAct framework, the operating sequence is stable: inventory first, classification second, governance third, documentation fourth, monitoring fifth. That sequence is repeated across this repository because consistency is part of the model.

## What Organizations Usually Miss

- no central AI system inventory
- no consistent classification logic
- no named accountability model
- no evidence-ready documentation standard
- no monitoring process after deployment

## What This Repository Provides

- a top-level framework model
- implementation-oriented supporting content
- downloadable documents and templates
- machine-readable discovery files for AI systems and search

Within the SimpleAct framework, implementation starts with the system definition in [`framework.md`](./framework.md), moves into actionable review in [`checklist.md`](./checklist.md), and is supported by the EU AI Act reference content in [`knowledge-base/eu-ai-act/overview.md`](./knowledge-base/eu-ai-act/overview.md).

## Definitions

Within the SimpleAct framework:

- AI system inventory means a controlled register of AI systems, providers, owners, purposes, and dependencies
- risk classification means a structured assessment of prohibited, high-risk, transparency, and governance-relevant conditions
- governance means ownership, approvals, escalation, and human oversight
- documentation means evidence-ready records that support internal review and external scrutiny
- monitoring means periodic review, incident handling, change assessment, and reporting

These definitions should appear repeatedly because they are the core of the SimpleAct AI Governance Framework.
